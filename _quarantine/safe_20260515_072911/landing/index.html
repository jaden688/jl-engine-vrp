Capabilities

Not another wrapper framework.

SparkByte models conversation state as a dynamic behavioral machine before the LLM ever sees the message.

⚙

Self-Forging Tools

Your agent writes, tests, and persists new tools at runtime via forge_new_tool. No redeploy. No restart. Live Julia eval with automatic persistence.

★

Behavioral State Machine

20-state grid (5 intensity x 4 control) models mood, drift pressure, and emotional aperture. Agents built on SparkByte don't feel like chatbots.

⚡

MCP Bridge

Claude Code, Cursor, Gemini CLI, Codex can discover and query your engine's state, memory, tools, and thoughts instantly via MCP.

🚀

A2A Protocol

Full Agent-to-Agent HTTP endpoint with JSON-RPC 2.0. Other agents can send tasks, check status, and get results. Standard compliant.

📋

Real OS Access

File I/O, shell commands, Playwright browser, GitHub, Bluetooth, SMS. Not sandboxed. Your agent operates at OS level.

🎭

5 Agents

SparkByte, Slappy, The Gremlin, Temporal, Supervisor. Each with unique emotional palettes, drive types, and behavioral profiles. Switch on the fly.

Quick start

# Clone and run
git clone https://github.com/jaden688/JL_Engine-SB.Omni.git
cd JL_Engine-SB.Omni
cp .env.example .env    # add your API keys
julia sparkbyte.jl      # UI at http://localhost:8081, A2A at :8082

# Or with Docker
docker compose up --build
