# Summary: Local AI Needs to be the Norm

The article argues that developers are over-relying on cloud-based AI APIs (like OpenAI/Anthropic) for features that could—and should—be handled locally on the user's device. 

Key points:
- **Fragility & Privacy:** Cloud-dependent features break when servers go down and raise significant privacy/data retention concerns.
- **Complexity:** Relying on external APIs turns simple UX features into complex, costly distributed systems.
- **Local AI Potential:** Modern devices have powerful, underutilized Neural Engines. Local AI is ideal for "data transformation" tasks (summarizing, classifying, extracting) on user-owned data.
- **Engineering Improvement:** Using local models with structured, typed outputs (e.g., Swift structs) makes AI features more predictable and reliable compared to parsing unstructured text.
- **Conclusion:** "AI everywhere" isn't the goal; useful, trustworthy software is. Developers should reserve cloud models for tasks that truly require them and keep the rest local.
